初回セットアップ
GitHubからMacにダウンロードして使えるようにする
1
ターミナルを開く
▼
Macに標準搭載されているターミナルアプリを開きます。
💡キーボードで Cmd + Space を押して「ターミナル」と入力 → Enterで起動
2
Documentsフォルダに移動する
▼
以下のコマンドをコピーしてターミナルに貼り付け、Enterを押してください。
ターミナル 
cd ~/Documents
3
GitHubからダウンロード（クローン）する
▼
「ユーザー名」の部分をあなたのGitHubユーザー名に変えて実行してください。
ターミナル 
git clone https://github.com/ユーザー名/glossary-app.git
⚠️「ユーザー名」はGitHubのアカウント名に変えてください。GitHubのリポジトリページのURLで確認できます。
4
フォルダの中に入る
▼
ターミナル 
cd glossary-app
5
必要なツールをインストールする
▼
Excelを読み込むために必要なツールをインストールします。初回のみ必要です。
ターミナル 
npm install
💡「added XX packages」と表示されれば成功です。警告（warn）が出ても問題ありません。
6
VS Codeで開く
▼
ターミナル 
code .
⚠️「command not found: code」と表示された場合は、VS Codeを開いて「Cmd+Shift+P」→「Shell コマンド: PATH内に 'code' コマンドをインストール」を実行してください。
7
管理画面を開いて動作確認する
▼
ターミナル 
open admin/admin.html
✅ブラウザで管理画面が開けば初回セットアップ完了です！

トラブルシューティング
うまくいかない場合の対処法
 「command not found: code」と表示された
VS Codeのcodeコマンドが設定されていません。
VS Codeを開いて Cmd+Shift+P →「Shell コマンド: PATH内に 'code' コマンドをインストール」を実行してください。その後ターミナルを新しく開き直してから再実行してください。
 「quote>」と表示されて止まった
クォート記号（' や "）が途中で入力された状態です。
キーボードで Ctrl + C を押してキャンセルし、もう一度コマンドをコピーして実行してください。
 Excelが読み込めない
ファイルがxlsx形式で保存されているか確認してください。
Libre Officeで「名前を付けて保存」→ファイル形式を「xlsx」に選択して保存し直してください。
 翻訳ボタンが動作しない
APIキーが正しく入力されているか確認してください。
APIキーは sk-ant-api03- で始まる文字列です。
console.anthropic.com で新しいキーを作成してください。
 GitHubにPushできない
以下を実行してGitの設定を確認してください：
ターミナル
git config --global user.name "あなたの名前"
git config --global user.email "メールアドレス"
 node_modulesのエラーが出る
以下を実行して必要なツールを再インストールしてください：
ターミナル
cd ~/Documents/glossary-app
npm install
 画像が表示されない
以下を確認してください：
① ファイル名のルール（例: 001_1.jpg）になっているか
② images/ と docs/images/ の両方にコピーされているか
③ ExcelのM列に正しいファイル名が入力されているか
